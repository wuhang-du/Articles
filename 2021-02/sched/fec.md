# 03-02 FEC方案的思考  

一图胜千言。 
```go

当前的模式：
skjA: |----1M----|----1M----|----1M----|  总共3M文件
skjB: |----1M----|----1M----|----1M----|  总共3M文件

问题：为了每个机器上都能随时补任何片，每个机器都部署了整份文件。

新模式：
skjA: |--9piece--|--9piece--|--9piece--|  总共27piece.
skjB: |--9piece--|--9piece--|--9piece--| 

每1M编码成16个piece.每个机器存9/16个，这样节省7/16的空间。
但是增加了cpu编解码的开销。
```
